Daily Multi-Tenant Paper Discovery & Digest Platform

Developed a scalable multi-tenant SaaS platform that helps individuals and research teams efficiently discover, rank, and digest scientific literature. Built on a Next.js and Supabase architecture, the system supports personalized workspaces with role-based access and delivers daily paper recommendations tailored to user-defined research preferences.

The platform implements an end-to-end paper pipeline that continuously ingests publications from arXiv and OpenAlex, removes duplicates, generates embeddings in batches, and ranks results using cosine similarity against weighted signals such as keywords, authors, and titles. An embedding-driven relevance mode enables more semantically meaningful search compared to traditional keyword filtering.

The application features a responsive dashboard with advanced search, filtering, pagination, vocabulary expansion, and persistent interface settings, alongside bilingual support (EN/ZH) and dark/light themes. Sensitive credentials are secured using AES-256-GCM encryption, and database workflows are automated through Supabase migrations with end-to-end testing to ensure reliability.

RSheet - A Multi-User Concurrent Spreadsheet System

RSheet is a concurrent spreadsheet server written in Rust that enables multiple clients to interact with a shared spreadsheet through live GET/SET operations and spreadsheet-style expressions. The system evaluates formulas by resolving cell dependencies including single-cell and range references, and automatically propagates updates to affected cells.

The architecture emphasizes correctness and thread safety. A background worker thread handles dependency updates to ensure consistent recalculation without blocking client requests. Recursive resolution and cycle detection were implemented to support complex formula chains while preventing invalid dependency graphs.
